# 2026-05-10

- Product/security requirement: one-time links should use an un-guessable ID/token.
- One-time links should automatically expire after 30 days (assumed: 30 days from issuance/creation).
- Idea: optionally support a “sliding” window / configurable expiry:
  - Issuer-selectable TTL (e.g., 7/14/21/30 days) as a preference.
  - Resend/refresh should re-issue a new token and revoke the prior unused token.
  - If we also “slide” on recipient access/click, be explicit in UX and consider a hard max-age cap + rate limits + audit logging to avoid unbounded extension.
    - Confirmed: hard max age cap = 90 days; extend-on-click only if link not yet used/completed.
